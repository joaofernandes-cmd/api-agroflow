WebAPI — BrPec Agropecuária

Endpoints são pontos de entrada por meio dos quais o sistema se comunica internamente e com clientes externos, associados a um verbo HTTP (GET, POST, PATCH, DELETE) que define a intenção de cada requisição. A documentação abaixo cobre todos os endpoints implementados no backend, ordenados conforme a prioridade dos Requisitos Funcionais mapeados no Artefato 1.

Base URL http://localhost:3000
RF001 — Alta

Movimentações do Rebanho

POST /movimentacoes Registrar uma nova movimentação

Descrição

Registra uma nova movimentação do rebanho (nascimento, morte, transferência, compra ou venda). Valida campos obrigatórios conforme o tipo da movimentação, conforme RN01.

Autenticação

Nenhuma (acesso por perfil de campo)

Body (application/json)

{
  "retiro_id":   string UUID  (obrigatório),
  "capataz_id":  string  (obrigatório),
  "tipo":        "nascimento" | "morte" | "transferencia" | "compra" | "venda"  (obrigatório),
  "estagio_vida": string (obrigatório),
  "origem":      string  (condicional: transferencia, nascimento, morte),
  "destino":     string  (condicional: transferencia),
  "quantidade":  number  (condicional: compra, venda, transferencia, nascimento),
  "causa_obito": string  (condicional: morte),
  "sincronizado": boolean
}

Respostas

StatusDescrição
201 CreatedMovimentação criada. Retorna o objeto criado.
400 Bad Request{ "error": "Campos obrigatorios nao informados" }
400 Bad RequestCampo inválido para o tipo informado.
GET /movimentacoes Listar todas as movimentações

Descrição

Retorna todas as movimentações cadastradas. Quando parâmetros de filtro são informados (retiro, tipo, status, dataInicio, dataFim), executa o filtro automaticamente.

Query Parameters (opcionais — ativam filtro)

ParâmetroTipoDescrição
retironumberID do retiro
tipostringTipo(s) separados por vírgula
statusstringpendente | validado
dataIniciostring (ISO)Data de início do período
dataFimstring (ISO)Data de fim do período

Respostas

StatusDescrição
200 OKArray de movimentações.
500 Internal Server ErrorErro interno.

Parâmetros de Rota

RF005 — Alta

Autenticação

POST /usuarios/login Autenticar usuário e obter token JWT

Descrição

Valida login e senha do usuário. Retorna o token JWT utilizado por todas as rotas protegidas. Capataz não recebe token por este endpoint (acesso de campo é por outro fluxo), conforme RN05/RN06.

Body (application/json)

{
  "login": string (obrigatório),
  "senha": string (obrigatório)
}

Respostas

StatusDescrição
200 OK
{ "usuario": { ...dados sem senha_hash }, "token": "JWT..." }
400 Bad RequestLogin ou senha ausentes.
401 UnauthorizedCredenciais inválidas.
403 ForbiddenUsuário inativo ou perfil Capataz.
500 Internal Server ErrorErro interno.

RF009 — Média

Filtros de Movimentações

GET /movimentacoes/filtrar Filtrar movimentações com critérios específicos

Descrição

Rota dedicada ao filtro de validação do Supervisor, conforme RN09. Permite seleção múltipla de tipos e status, mas apenas um retiro por vez. Quando nenhum status é informado, retorna pendente por padrão.

Query Parameters

ParâmetroTipoObrigatórioDescrição
retironumberSimID do retiro
tipostring(s) separados por vírgulaNãonascimento, morte, transferencia, compra, venda
statusstring(s) separados por vírgulaNãopendente, validado (default: pendente)
dataIniciostring (ISO 8601)NãoInício do período
dataFimstring (ISO 8601)NãoFim do período (inclui horário 23:59:59)

Respostas

StatusDescrição
200 OKArray de movimentações filtradas.
400 Bad RequestRetiro ausente ou data inválida.
500 Internal Server ErrorErro interno.
GET /movimentacoes/pendentes Listar movimentações pendentes de validação

Query Parameters (opcional)

ParâmetroTipoDescrição
retiroIdstring UUIDFiltrar por retiro específico

Respostas

StatusDescrição
200 OKArray de movimentações com status pendente.
